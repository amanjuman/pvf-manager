1
Copy or download the script, then upload to your host

        
# From your local machine:
scp iptables-portforward.sh root@your-host-ip:/etc/iptables-portforward.sh

# Or paste it directly via SSH:
nano /etc/iptables-portforward.sh

      

      

        
2
Make it executable

        
chmod +x /etc/iptables-portforward.sh

      

      

        
3
Create the systemd service

        
cat > /etc/systemd/system/iptables-portforward.service << 'EOF'
[Unit]
Description=IPTables Port Forwarding Rules for SDN NAT VMs
After=network.target pve-firewall.service

[Service]
Type=oneshot
ExecStart=/etc/iptables-portforward.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
EOF

      

      

        
4
Enable and start

        
systemctl daemon-reload
systemctl enable --now iptables-portforward.service

      

      

        
5
Verify rules are active

        
systemctl status iptables-portforward.service
iptables -t nat -L PREROUTING -n -v --line-numbers

      

      

        
6
After adding/removing rules — reapply

        
# After updating the script with new rules:
systemctl restart iptables-portforward.service

# After Proxmox SDN Apply (which flushes iptables):
systemctl restart iptables-portforward.service

# Check active rules at any time:
iptables -t nat -L PREROUTING -n -v

      

    

  


  
  

    

      
What is this?

      
A browser-based tool to manage iptables DNAT port forwarding rules for Proxmox VE hosts running SDN Simple zones with NAT. It generates a ready-to-use shell script and systemd service you can drop onto your host.

      
All data is stored in your browser's localStorage. Nothing is sent to any server.

    

    

      
How it works

      
When your VMs or LXCs sit behind a Proxmox NAT zone, they share your host's public IP. Port forwarding uses iptables DNAT to redirect specific host ports to the correct internal VM/LXC IP and port.

      
The generated script flushes and reapplies all PREROUTING/FORWARD rules on each run, so it's safe to call via systemctl restart without duplicating rules.

    

    

      
Proxmox SDN requirement

      
This tool is designed for Proxmox SDN simple zones with snat 1 enabled. Your WAN bridge (e.g. vmbr0) must be the interface facing the internet.
